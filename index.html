Formulários

Formulários são a conexão entre o front end e o backend. É onde ocorre a interação com o usuário.No google, por exemplo, ele ve as palavras que o usuário está passando no campo de pesquisa e a página retorna com a nossa resposta.

Para criar um formulário usamos a tag FORM. Quando vc clica no botão enviar, todos os campos dentro desse formulário serão enviados para o backend, que retornará com a resposta à solicitação.

Os métodos de envio pode ser GET ou POST. E precisamos de uma ACTION (O endereço que vc vai utilizar para criar uma requisição). Se o seu front end estiver rodando no mesmo domínio que o seu web service, não precisa por o endereço completo do domínio (http://...) mas somente "/endereço do endpoint". Como exemplo, se o endereço for www.patyprogramadorabr.com e o backend estiver em www.patyprogramadorabr.com/cadastro, só precisa colocar /cadastro.

INPUTS

Campo INPUT TYPE TEXT é usado para campos em que usuário irá DIGITAR alguma informação solicitada, como NOME ou E-MAIL.

O NAME serve para que o navegador diferencie os diferentes inputs que existirem no formulário (nome e email, por exemplo).

O campo SUBMIT não precisa de nome (somente INPUT TYPE="SUBMIT")

No Browser:

Nome: E-mail:

Experimente colocar dados aleatórios neste formulário, sem ter uma página pronta para receber esses dados num backend, ele nao executa pq a página /cadastro não existe. Se você reparar, ao enviar os dados, aparece toda a informação digitada na barra de endereço. Para os dados não ficarem expostos, temos q usar o METHOD="POST".

Como exemplo, em uma página de login, não é recomendável que o nome de usuário e senha fiquem expostos. Para que isso não aconteça, utilizamos o METHOD:"POST". o que aparece no nosso endereço é
http://127.0.0.1:5500/cadastro?email=paty&nome=paty
Analisando o que está escrito no endereço, vemos que o que está antes da interrogação (cadastro) é a nossa ACTION (é onde o backend estaria) Nosso primeiro input é EMAIL. depois do igual temos o email que foi digitado. Nosso segundo input é nome. Depois do igual temos o nome que foi digitado

>

Como método POST vemos que após digitar os dados, a página não mostra na sua barra de endereço o que foi digitado. Como dissemos anteriormente, é ideal para páginas de login e senha.

YOUTUBE

Vamos ao YOUTUBE. Digitando programação no campo de busca aparece assim:
https://www.youtube.com/results?search_query=programa%C3%A7%C3%A3o

Analisando o caminho que aparece no endereço já entendemos que o método é GET, pois os dados estão visíveis. Antes da primeira interrogação temos "https://www.youtube.com/results", que é a nossa ACTION. Sabemos que é um input do tipo text, pois temos que digitar no campo de busca para receber o resultado. Logo depois da nossa action temos "search_query", que é o NAME
